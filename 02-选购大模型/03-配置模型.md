# 2.3 配置 OpenClaw 使用模型

**在服务器上配置 API Key，让 OpenClaw 调用 AI**

---

## 📁 配置文件位置

OpenClaw 配置文件在：`~/.openclaw/config.yaml`

---

## ⚙️ 配置 MiniMax

用 vim 或 nano 编辑：

```bash
nano ~/.openclaw/config.yaml
```

填入以下内容：

```yaml
providers:
  minimax:
    api_key: "你的API Key"
    model: "MiniMax-M2"

default_provider: minimax
```

**保存方法：** Ctrl+X → Y → Enter

---

## ✅ 验证配置

重启 OpenClaw：

```bash
openclaw gateway restart
```

或者启动后测试对话：

```bash
openclaw chat
```

能正常回复即表示配置成功。

---

## 🔄 切换不同模型

如果同时配置了多个模型：

```yaml
providers:
  minimax:
    api_key: "你的MiniMax Key"
    model: "MiniMax-M2"
  deepseek:
    api_key: "你的DeepSeek Key"
    model: "deepseek-chat"

default_provider: minimax   # 默认使用 MiniMax
```

对话时指定模型：
```bash
openclaw chat --provider deepseek
```

---

## ❓ 常见问题

**Q：提示 `Invalid API Key`？**
→ 检查 Key 是否正确，注意不要有多余空格。

**Q：提示 `Insufficient Balance`？**
→ 余额不足，去 MiniMax 控制台充值。
