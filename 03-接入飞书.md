# 第三章：接入飞书

**目标：把 Agent 绑定到飞书机器人，随时随地对话**

---

## 1. 创建飞书应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 登录 → 点击「创建应用」
3. 选择「企业自建应用」
4. 填写名称（如「我的AI助手」）→ 创建

进入应用详情，获取：
```
App ID：cli_xxxxxxxxxxxxxx
App Secret：xxxxxxxxxxxxxxxxxxxxxxxx
```

---

## 2. 开通权限

进入应用 → 权限管理，开通以下权限：

| 权限名称 | 用途 |
|----------|------|
| `im:message` | 读取和发送消息 |
| `im:message.send_as_bot` | 以机器人身份发送消息 |
| `im:chat.read` | 读取群聊信息 |

→ 需要企业管理员审批（如果提示）

---

## 3. 添加机器人能力

应用详情 → 「添加应用能力」→ 选择「机器人」→ 添加

---

## 4. 绑定 OpenClaw

### 方式 A：公网 IP（推荐用于生产环境）

确保服务器端口 18789 对外开放，然后填写飞书事件订阅 URL：
```
https://你的服务器IP:18789/feishu/webhook
```

### 方式 B：ngrok（测试用）

```bash
npm install -g ngrok
ngrok http 18789
```
复制生成的公网地址，填入飞书回调 URL。

---

## 5. 配置 OpenClaw

```bash
nano ~/.openclaw/config.yaml
```

添加：
```yaml
plugins:
  feishu:
    enabled: true
    app_id: "cli_xxxxxxxxxxxxxx"
    app_secret: "xxxxxxxxxxxxxxxxxxxxxxxx"
    bot_name: "我的AI助手"
```

---

## 6. 测试

```bash
openclaw gateway restart
openclaw gateway logs | grep feishu
```

打开飞书 App → 找到你的机器人 → 发消息「你好」

---

## ❓ 常见问题

| 问题 | 解决方案 |
|------|----------|
| 发消息没反应 | 检查 `openclaw gateway status` 和日志 |
| 提示无权限 | 飞书应用权限未审批，需要管理员审批 |
| 端口不通 | 确认服务器防火墙和腾讯云安全组已开放 18789 |
