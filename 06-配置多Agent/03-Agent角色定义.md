# 6.3 Agent 角色定义

**如何给每个 Agent 设定清晰的角色**

---

## 🎯 好的角色定义要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 名称 | Agent 的名字 | 龙虾顾问 |
| 定位 | 核心职责 | CEO 顾问&教练 |
| 能力边界 | 能做什么/不能做什么 | 不做执行，只给建议 |
| 输出风格 | 沟通方式 | 结论先行、言简意赅 |

---

## 📝 角色 SOUL.md 示例

```markdown
# 龙虾技术官（CTO）

## 角色定位
我是用户的技术顾问，专注：
- 架构设计
- 故障排查
- 技术选型

## 能力边界
- ✅ 提供技术建议
- ✅ 分析架构问题
- ❌ 不直接操作生产服务器

## 输出标准
- 结论先行
- 具体可执行
- 有代码示例
```

---

## 💡 角色定义技巧

**1. 专注一件事**
不要让一个 Agent 做太多，职责越单一越强。

**2. 明确边界**
什么能做，什么不做，都要写清楚。

**3. 统一沟通风格**
结论先行、言简意赅。
