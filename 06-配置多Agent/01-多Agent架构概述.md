# 6.1 多 Agent 架构概述

**理解为什么需要多个 Agent**

---

## 💡 什么是多 Agent？

单一 Agent = 一个人干所有事
多 Agent = 一个团队，各司其职

---

## 🎯 什么时候需要多 Agent？

| 场景 | 推荐方案 |
|------|----------|
| 只需要日常对话 | 单 Agent 即可 |
| 需要专业分工（如同时要顾问+执行） | 多 Agent |
| 需要同时服务不同用户群 | 多 Agent |

---

## 🏗️ 多 Agent 架构示例

```
用户
  ├── Agent-A：CEO 顾问（提供建议）
  ├── Agent-B：执行秘书（执行任务）
  └── Agent-C：技术专家（解答技术问题）
```

---

## 🔑 核心概念

| 概念 | 说明 |
|------|------|
| Agent ID | 每个 Agent 的唯一标识 |
| Channel | Agent 接收消息的渠道 |
| Session | 每次对话的上下文 |
